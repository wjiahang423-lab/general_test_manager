## 软件简介

通用测试管理工具是一款用于 EOL（End of Line）产线测试的管理软件，支持：

- **测试计划管理**：通过配置文件定义完整的测试流程
- **数据驱动测试**：通过 YAML 文件或 Excel 表格导入大量测试用例
- **灵活配置**：测试脚本与测试计划分离，用户可灵活编辑

### 启动方式
双击 `ATETest.exe` 启动程序，启动后出现主界面。

## 项目管理

### 加载测试计划入口
1. 点击顶部菜单 **"测试计划"** → **"加载测试计划"**
2. 浏览并选择 `.yaml` 格式的测试计划文件
3. 加载成功后，可以在 **"测试计划"** 列表看到该计划

### 操作步骤说明
- 测试计划是测试流程的定义文件（`.yaml`格式）
- 测试用例数据存放在测试计划引用的 YAML/Excel 文件中
- 测试脚本在 `test_scripts` 目录中由测试人员编写

## 测试计划配置

测试计划采用 YAML 格式，主要结构如下：

### 基本结构

```yaml
# 测试计划配置示例
name: 测试计划名称              # 计划名称
version: '1.0'                 # 计划版本号
description: 测试描述           # 可选的描述信息

# 全局参数（所有步骤共享）
global_params:
  station_id: LINE-ACU-01
  ec_sn: ECU123456
  timeout_base: 30.0

# 全局变量
variables: {}
variable_types: {}

# 测试序列（测试步骤的顺序）
sequences:
- name: 初始化序列              # 序列名称
  skip: false                   # 是否跳过整个序列（false=执行）
  steps:                        # 序列中的步骤
    - name: 步骤 1
      step_type: script         # 脚本步骤 | measurement | delay
      script: 脚本文件名.py      # test_scripts 目录下的 Python 文件
      function: 函数名           # 要调用的函数名
      params: {}                # 传递给函数的参数
      limits:                   # 可选：测量值上下限
        low: 10.0
        high: 20.0
        unit: V
      on_fail: continue         # 失败后：continue（继续）| abort（终止）
      timeout: 30.0             # 超时时间（秒）
      retry_count: 1            # 失败后重试次数
      breakpoint: false         # 是否断点
      return_map: {}            # 返回映射：{返回值键：变量名}
      loop_source:             # 如果是循环步骤
        test_items.yaml        # 循环数据源
        key: key_name          # 循环数据表名
        loop_item_type: measurement  # item 类型：script|measurement
```

### 示例：延期步骤

```yaml
- name: 延时 3 秒
  step_type: delay
  params:
    seconds: 3.0
  on_fail: abort
  timeout: 30.0
```

## 数据驱动测试用例

### YAML 数据驱动

测试数据使用 YAML 文件组织，通过 `loop_source`在测试计划中引用。

#### 测量型测试用例（measurement）

```yaml
# test_items.yaml
fire_loop:
  - name: "点火回路 1 阻值测试"
    var_name: "DvEol_ObsResIn_B1B2Resdata_f32"
    min: 3.0
    max: 5.0
    unit: Ohm
  - name: "点火回路 2 阻值测试"
    var_name: "DvEol_ObsResIn_B3B4Resdata_f32"
    min: 3.0
    max: 5.0
    unit: Ohm
```

测试脚本函数签名：
```python
def measure(params):
    val = 4.5  # 实测值
    # 自动判断传入的 min/max 参数
    if min is not None and val < min:
        return {"pass": False, "message": f"值{val}低于最小值{min}"}
    if max is not None and val > max:
        return {"pass": False, "message": f"值{val}高于最大值{max}"}
    return {"pass": True, "value": val, "unit": "Ohm"}
```

#### 脚本型测试用例（script）

```yaml
dic_write:
  - did_name: "F187"
    var_name: "did_rw_flag"
    length: 15
    expected: "LAJ-56040012"
  - did_name: "F18A"
    var_name: "did_flag"
    length: 3
    expected: "BOS"
```

测试脚本函数签名：
```python
def run(params):
    # params 包含：
    #   - did_name: "F187"
    #   - var_name: "did_rw_flag"
    #   - length: 15
    #   - expected: "LAJ-56040012"
    success = write_did_and_verify(...)
    return {
        "pass": success,
        "message": "DID 写入成功" if success else f"DID {params['did_name']}写入失败"
    }
```

#### 特殊测试用例键

```yaml
# 开关控制类检测
smi970:
  - name: HfAx
    var_name: "DvEol_ObsIMUIn_SmiAccelerateHfAxdata_f32"
    min: -0.1
    max: 0.1

# DID 读取验证
did_checks:
  - did_name: "F187"
    var_name: "VehicleManufacturerSparePartNumId0xF187"
    length: 15
    expected: "LAJ-56040012"
```

#### 短信/定长类检测

```yaml
# 定长字符串检测（int 类型）
SMA760:
  - name: "SMA760 有效性检测"
    var_name: "DvEol_ObsIMUIn_Sma760DataValid_b"
    length: 1
    expected_int: 0      # 使用 expected_int 进行整数比较
```

## 导入测试用例

### 使用 Excel 导入

本工具支持通过 Excel 格式导入测试用例，文件需放置在 `test_scripts`目录下的对应模块文件夹中。

#### Excel 模板格式

1. **打开模板文件**：`Excel 测试用例模板.xlsx`
2. **根据测试类型填写对应的 sheet**
3. **保存并关闭**后，可在测试计划中引用

详见 [Excel 测试用例文件说明.docx](Excel 测试用例文件说明.docx)

## 运行测试

### 启动测试

1. 加载测试计划后，在测试计划列表中选中该计划
2. 点击 **"运行"** 按钮
3. 在运行面板中查看实时执行进度和日志输出

### 测试结果

- **PASS**: 所有测试项通过
- **FAIL**: 存在失败项
- **ABORT**: 被中止

测试日志实时显示，超时/重试/断点状态均会记录。

## 常见配置示例

### 完整测试计划示例

```yaml
name: ACU 产线 EOL 测试
version: '1.0'
description: 安全气囊控制器产线测试
global_params:
  station_id: LINE-ACU-01

sequences:
- name: 初始化
  skip: false
  steps:
    - name: 上电并进入 EOL 模式
      step_type: script
      script: acu/setup_eol.py
      function: setup
      on_fail: abort

- name: 点火回路阻值测试（24 项数据驱动）
  skip: true                                    # true=跳过此序列
  steps:
    - name: 点火回路阻值
      step_type: loop                           # 循环类型
      script: acu/measure_fire_loop_item.py
      function: measure
      loop_source: acu/test_data/test_items.yaml
      loop_key: fire_loop                       # 引用表名
      loop_item_type: measurement               # 数据项类型
      on_fail: continue
      timeout: 15.0
```

## 附录

### A. 脚本错误排查
- 查看运行面板的日志输出（黑框区域会通过 PyInstaller 输出合并到 UI）
- 检查测试脚本是否有语法错误
- 确认脚本中存在 `pass True/False` 或者返回值不符合数据驱动的要求

### B. 支持的文件格式
- 测试计划：`.yaml`
- 测试数据：`.yaml` 或 Excel 模板
- 测试脚本：`.py`
